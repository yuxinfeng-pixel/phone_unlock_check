Android APK 编译安装完全指南
方法一：使用 Android Studio（推荐，最简单）
第一步：下载安装 Android Studio
1. 
访问官网下载：https://developer.android.com/studio
2. 
下载对应你电脑系统的版本（Windows/Mac/Linux）
3. 
按照安装向导完成安装
Windows 用户注意：安装路径不要有中文或空格，建议  C:\Android\Studio 
第二步：创建新项目并导入代码
1. 
打开 Android Studio → 点击 "New Project"
2. 
选择模板：Empty Views Activity（注意不是 Empty Activity）
3. 
填写项目信息：
 
Name:  PhoneCheck 
 
Package name:  com.phonecheck.app 
 
Language:  Kotlin 
 
Minimum SDK:  API 24: Android 7.0 
 
Build configuration language:  Kotlin DSL (build.gradle.kts) 
4. 
点击 Finish，等待项目初始化完成（首次可能需要下载组件，耐心等待）
第三步：替换/创建文件
按照以下目录结构，把代码复制到对应位置：

具体操作：
1. 替换  app/build.gradle.kts ：
删除原有内容，粘贴我提供的 build.gradle 内容（注意把 Groovy 语法转为 Kotlin DSL）：
kotlin
plugins {
    alias(libs.plugins.android.application)
    alias(libs.plugins.kotlin.android)
}

android {
    namespace = "com.phonecheck.app"
    compileSdk = 34

    defaultConfig {
        applicationId = "com.phonecheck.app"
        minSdk = 24
        targetSdk = 34
        versionCode = 1
        versionName = "1.0"
    }

    buildFeatures {
        viewBinding = true
    }

    buildTypes {
        release {
            isMinifyEnabled = false
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "proguard-rules.pro"
            )
        }
    }
    compileOptions {
        sourceCompatibility = JavaVersion.VERSION_17
        targetCompatibility = JavaVersion.VERSION_17
    }
    kotlinOptions {
        jvmTarget = "17"
    }
}

dependencies {
    implementation(libs.androidx.core.ktx)
    implementation(libs.androidx.appcompat)
    implementation(libs.material)
    implementation(libs.androidx.constraintlayout)
}

2. 修改  settings.gradle.kts  确保包含依赖库：
打开  gradle/libs.versions.toml ，确认有以下内容（通常默认就有）：
toml
[versions]
agp = "8.3.0"
kotlin = "1.9.22"
coreKtx = "1.12.0"
appcompat = "1.6.1"
material = "1.11.0"
constraintlayout = "2.1.4"

[libraries]
androidx-core-ktx = { group = "androidx.core", name = "core-ktx", version.ref = "coreKtx" }
androidx-appcompat = { group = "androidx.appcompat", name = "appcompat", version.ref = "appcompat" }
material = { group = "com.google.android.material", name = "material", version.ref = "material" }
androidx-constraintlayout = { group = "androidx.constraintlayout", name = "constraintlayout", version.ref = "constraintlayout" }

[plugins]
android-application = { id = "com.android.application", version.ref = "agp" }
kotlin-android = { id = "org.jetbrains.kotlin.android", version.ref = "kotlin" }

3. 创建 Kotlin 文件：
在  app/src/main/java/com/phonecheck/app/  目录下：
 
右键 → New → Kotlin Class/File → 输入文件名（如  MonitorService ）→ 选择 Class
 
把我提供的代码粘贴进去
4. 替换 XML 文件：
打开  res/layout/activity_main.xml ，删除原有内容，粘贴我提供的布局代码。
第四步：同步项目
点击 Android Studio 顶部工具栏的 "Sync Now"（大象图标），或按  Ctrl+Shift+O （Windows）/  Cmd+Shift+O （Mac）。
等待 Gradle 同步完成，底部状态栏显示 "Sync finished"。
第五步：编译生成 APK
方式 A：调试版 APK（快速测试）
1. 
点击菜单栏 Build → Build Bundle(s) / APK(s) → Build APK(s)
2. 
等待编译完成
3. 
右下角弹出提示，点击 locate 找到 APK 文件
4. 
路径通常在： app/build/outputs/apk/debug/app-debug.apk 
方式 B：正式签名 APK（推荐用于长期使用）
1. 
点击菜单栏 Build → Generate Signed Bundle / APK...
2. 
选择 APK，点击 Next
3. 
点击 Create new... 创建密钥库：
 
Key store path: 选择保存位置，如  C:\Users\你的用户名\phonecheck.jks 
 
Password: 设置密码（记住它！）
 
Key alias:  phonecheck 
 
Key password: 可以和上面一样
 
Validity (years): 25
 
Certificate: 至少填一个名字，如  CN=YourName 
4. 
点击 OK → Next
5. 
选择 release，勾选 V1 (Jar Signature) 和 V2 (Full APK Signature)
6. 
点击 Finish
7. 
生成的 APK 在： app/release/app-release.apk 
第六步：安装到手机
方法一：USB 数据线安装（推荐）
1. 
开启手机开发者模式：
 
设置 → 关于手机 → 连续点击"版本号"7次 → 提示"您已处于开发者模式"
2. 
开启 USB 调试：
 
设置 → 系统 → 开发者选项 → 开启 "USB 调试"
3. 
连接电脑：
 
用数据线连接手机和电脑
 
手机上选择 "传输文件"（不要选仅充电）
 
允许电脑调试（手机上会弹窗，点击"允许"）
4. 
在 Android Studio 中安装：
 
点击顶部工具栏的绿色三角形 "Run"（或按  Shift+F10 ）
 
选择你的手机设备 → OK
 
等待安装完成，手机会自动打开应用
方法二：手动传输安装
1. 
把生成的 APK 文件复制到手机（微信、QQ、数据线、U盘都可以）
2. 
在手机上找到 APK 文件
3. 
点击安装，如果提示"未知来源"，去设置里允许
4. 
安装完成



方法二：命令行编译（适合高级用户）
如果你不想用 Android Studio，可以用命令行：
1. 安装命令行工具
下载 Android SDK Command Line Tools
解压到  C:\Android\Sdk\cmdline-tools\latest\ 
2. 配置环境变量
在系统环境变量  PATH  中添加：

C:\Android\Sdk\cmdline-tools\latest\bin
C:\Android\Sdk\platform-tools

3. 安装必要组件
bash
sdkmanager "platforms;android-34"
sdkmanager "build-tools;34.0.0"
sdkmanager "platform-tools"

4. 创建项目结构
手动创建文件夹结构，把所有代码文件放到对应位置。
5. 编译
bash
# 进入项目根目录
cd PhoneCheck

# 赋予 gradlew 执行权限（Mac/Linux）
chmod +x gradlew

# 编译调试版
./gradlew assembleDebug

# 或编译正式版（需要配置签名）
./gradlew assembleRelease

APK 输出在  app/build/outputs/apk/



方法三：在线编译工具（最简单，无需安装任何东西）
如果你电脑配置不够或不想折腾，可以用在线工具：
使用 GitHub Actions 自动编译
1. 
注册 GitHub 账号
2. 
创建新仓库，上传所有代码
3. 
创建文件  .github/workflows/build.yml ：
yaml
name: Build APK

on:
  push:
    branches: [ main ]
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
          
      - name: Setup Android SDK
        uses: android-actions/setup-android@v3
        
      - name: Grant execute permission for gradlew
        run: chmod +x gradlew
        
      - name: Build with Gradle
        run: ./gradlew assembleDebug
        
      - name: Upload APK
        uses: actions/upload-artifact@v4
        with:
          name: app-debug
          path: app/build/outputs/apk/debug/app-debug.apk

4. 
提交代码后，GitHub 会自动编译
5. 
在 Actions 页面下载编译好的 APK


常见问题解决
表格
问题
解决方法
Gradle 同步失败
检查网络，尝试切换镜像源（阿里云镜像）
找不到  viewBinding 
确认  buildFeatures { viewBinding = true }  已添加
安装时解析包错误
确保 minSdk 不大于你手机的 Android 版本
短信发送失败
检查是否授予了 SEND_SMS 权限
服务被杀后台
去系统设置关闭电池优化，允许自启动
编译提示 JDK 版本不对
在 Android Studio → File → Settings → Build → Gradle 中设置 JDK 17

Android Studio 已安装
 
所有 Kotlin 文件已创建在正确路径
 
XML 布局文件已替换
 
 build.gradle.kts  已更新
 
Gradle 同步成功，无报错
 
手机已开启 USB 调试
